## 今月末までの実用化方針

仲間内で使う試験運用は、Google Sheetsを共有データ置き場にする想定で進めます。

想定構成:

```text
GitHub Pages
+ Google Apps Script
+ Google Sheets
```

まずはローカル保存のUI/運用フローを磨き、次にGoogle Sheets同期を追加します。

本格的な多人数/長期運用が必要になった場合は、Supabase Auth + Supabase Postgresへの移行を検討します。

## ログイン仕様

現在はローカル保存の簡易アカウントです。

- 初回登録ユーザーはAdmin
- 2人目以降はmember
- ログイン中ユーザーの表示名を現在参加者として扱う
- パスワード平文は保存しない
- Web Crypto APIのPBKDF2で派生ハッシュを保存

注意:

本物の複数端末共有アカウントではありません。友達全員が同じデータを共有する本番運用にするなら、Supabase AuthやFirebase Authなどへの置き換えが必要です。
